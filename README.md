🎯 О проекте
Репозиторий создан для демонстрации подходов к системному программированию на Go. Основной фокус направлен на работу с I/O, конкурентность и экономное потребление ресурсов сервера.

Здесь реализовано:
🚀 Streaming I/O: Использование io.TeeReader и io.Copy защищает систему от переполнения памяти
♻️ On-the-fly Deduplication: Расчет SHA-256 суммы файла прямо во время потоковой записи на диск
🗃️ Metadata WAL: Потокобезопасная in-memory база метаданных на sync.RWMutex со сбросом состояния в JSON
🧹 Background GC: Фоновая сборка мусора для автоматического удаления файлов-сирот
🛑 Graceful Shutdown: Корректное завершение HTTP-запросов и остановка при получении системных сигналов

Цель проекта: Показать архитектурные паттерны создания бэкенд-компонентов без использования сторонних фреймворков и библиотек.
📁 Структура репозитория

go-mini-cloud/
├── 📁 cmd/
│   └── 📁 server/         # Точка входа, настройка контекстов и запуск сервера
├── 📁 internal/

│   └── 📁 worker/         # Фоновые процессы (Garbage Collector)
├── 📄 go.mod              # Инициализация Go-модуля
└── 📄 README.md           # Документация проекта

🛠 Технологии
Go 1.21+ - Основной язык программирования (используется только стандартная библиотека)
.json - Формат хранения персистентных метаданных
net/http - Встроенная маршрутизация и обработка Range-запросов
sync / io / os - Низкоуровневая работа с примитивами синхронизации и потоками

⚙️ Требования и запуск
Для локального запуска
Проекту не нужны внешние СУБД или брокеры очередей. Требуется только установленный компилятор Go.

📥 Клонируйте репозиторий на локальную машину
📦 В корневой папке выполните go mod tidy для инициализации модуля
▶️ Запустите бэкенд командой go run ./cmd/server
📁 Сервер автоматически создаст директорию cloud_storage_data для хранения бинарных файлов и базы метаданных

💻 Использование API
Загрузка файла
Отправьте PUT-запрос, передав оригинальное имя через специальный заголовок:


curl -X PUT \
  --header "X-Original-Name: my_video.mp4" \
  -T local_file.mp4 \
  http://localhost:8080/upload
  
Скачивание файла
Отправьте GET-запрос с указанием полученного Hash ID. Сервер отдаст заголовок Content-Disposition, и файл скачается под своим оригинальным именем:
curl -O -J http://localhost:8080/download/<HASH_ID>
